Adding Danger

We have a character moving all over the place to be clicked on to win. We also need some danger so that we have a chance to lose.


In order to begin, we need to create a new scene for this bomb. Each scene can only hold one parent node. So we can’t make the bomb and all of its components in the UFO scene because it would mean the Bomb is part of the UFO and not a separate object altogether.


To create a new scene, click on the + button next to the UFO scene:



Click on the + symbol under the “Scene” tab to create the parent node:


Search for “Area2D” and click “Create”:



Rename the parent node to Bomb:

Press CTRL-S or File->Save Scene to save the scene and name it “Bomb”:

Right-click on the Bomb node and add a new child node:



Search and add the sprite node:


Do it again and add a CollisionShape2D



Click on the Sprite node:


Go to the “Inspector” view tab and change the sprite to the bomb sprite:



Click on CollisionShape2D:


Go to the “Inspector” view tab and change the “Shape” option (if you use the sprite provided, then you want the “Circle” shape):



Find your sprite in the main game window and change the collision box until it lines up with your sprite:


The last thing we need is to attach a script to the bomb!


Right-click on the Bomb node and “Attach Script”:


Accept the default values:


Delete the code given (except for the first line):



The code for the Bomb will be almost identical to what we did with the UFO.  So we can just copy and paste the UFO code to the Bomb script.


Click on “UFO.gd” to go back to the original script you wrote:


Highlight the code and right click and copy (or press CTRL-C):


Go back to the “Bomb.gd” script, and paste your code (right click and press paste, or CTRL-V):



There is only one section that we need to change. The input event for the bomb shouldn’t move the bomb to a new location, but instead disappear (because it blew up).


To do this, first change the name of the function “_on_UFO_input_event” to “_on_Bomb_input_event”. The only thing you’re changing is UFO to Bomb:



We need to set the signal of the input event, just like we did with the UFO.


First click on the Bomb node:


Go to the “Node” tab and double click the “input_event” signal:



Click “Connect”:


And it’s done!


Remember that we don’t want the bomb to move to a new location when left pressed, so we want to delete the code that is highlighted below:


Replace with a quick function call that will delete the node:


Quick note, there are two ways to delete a node. One is “queue_free” which completely and safely deletes the node. The other is “remove_child” which can be a little more dangerous (because it could cause errors in other parts of your code if you’re not careful) however it saves any data that you may have set for it (this is good if you just want to hide something, but bring it back later).


Let’s test! However, we can’t press the normal play button. Because it will still go to the scene with the UFO.
